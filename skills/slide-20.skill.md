# Slide 20: 商业合作与服务支持

## 页面逻辑与数据映射
- **主题**：把“服务支持”从简单三条 bullet，重构为一条完整的客户成功链路。
- **核心观点**：Rune 提供的不是单次交付，而是从售前咨询、架构设计，到驻场实施、定制开发，再到 7x24 支持与培训赋能的持续合作体系。
- **本页约束理解**：
  - 只继承全局白底科技感、浅网格、柔光背景与统一动效节奏。
  - 用户明确要求不要受既有页面版式影响，因此不套用前页母版，单独设计为“服务链路 + 支持中枢”页面。

## 布局体系 (Service Journey Only)
采用 **顶部标题 + 中央三段服务链路 + 底部结论条**：
1. **标题区**
   - 标题直接点明“商业合作与服务支持”。
   - 副标题明确这页不是讲单项服务，而是讲“项目成功如何被保障”。
2. **中部服务链路主画布**
   - 使用三张斜向错位卡片表达 `咨询 -> 交付 -> 持续运营`。
   - 去掉左右辅助说明卡，只保留三张主卡和背景路径，让阅读焦点完全落在服务阶段本身。
3. **底部结论条**
   - 收束为“Rune 提供的是覆盖方案共创、实施落地、运维陪跑、培训赋能的企业级合作体系”。

## 视觉风格
- **整体方向**：偏客户成功 / 服务设计页，而不是产品功能说明页。
- **背景**：
  - 维持白底、浅蓝网格、柔光与大面积浅色渐变。
  - 增加大尺寸圆弧与横向光带，弱化工程感，增强“服务流动”感。
- **组件语言**：
  - 三张核心服务卡采用轻旋转错位，避免传统横向并排结构。
  - 去掉外围大容器和两侧说明卡，让页面更像独立悬浮的服务旅程。

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题进入。
2. `0.26s`：服务链路主画布进入。
3. `0.38s`：第一张服务卡进入。
4. `0.52s`：第二张服务卡进入。
5. `0.66s`：第三张服务卡进入。
6. `0.88s`：底部结论条收尾。

## 页面目标
- 让客户第一眼理解 Rune 有完整的合作与服务体系，而不只是软件产品本身。
- 让销售或方案团队可以顺着本页清晰讲出：前期怎么咨询、中期怎么落地、后期怎么持续陪跑。
