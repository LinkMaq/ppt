# Slide 11: 开箱即用的应用市场

## 页面逻辑与数据映射
- **主题**：展示 Rune 应用市场如何把数据库、中间件、AI 应用和企业专属模板统一收敛，帮助用户零代码完成部署和后续运维。
- **核心观点**：应用市场不是单纯的模板列表，而是覆盖模板发现、参数配置、数据挂载、实例运维和模板沉淀的完整交付入口。
- **信息映射**：
  - 左侧主视觉：`Rune App Market` 截图式主界面，承载搜索、分类、指标和 6 张应用卡。
  - 应用卡内容：`Dify / Coze / ComfyUI / MySQL / MongoDB / JupyterLab Pro`
  - 右侧上半区：`一键部署路径` 四步说明，映射“选择模板 -> 配置参数 -> 挂载数据 -> 上线运维”
  - 右侧中部：`生命周期 / 数据管理 / 可观测性 / 模板沉淀` 四张价值卡
  - 右侧底部：差异化总结，强调 Rune 同时覆盖 AI 与容器场景
  - 底部结论：收束到“加速创新但不牺牲治理能力”

## 布局体系 (Marketplace Console + Delivery Rail)
采用 **顶部标题 + 中部左右双区 + 底部结论条**：
1. **顶部标题区**
   - 标题直接点题“开箱即用的应用市场”。
   - 副标题强调统一部署入口、零代码拉起和数据挂载能力。
2. **摘要条**
   - 用五个轻量标签概括页面关键词：海量模板、一键部署、全生命周期、统一数据管理、企业专属模板。
3. **左侧市场主界面**
   - 使用截图式控制台语言，包含搜索栏、分类筛选、统计指标和 3 x 2 应用卡网格。
   - 应用卡混合展示 AI 应用、数据库和企业开发模板，直接体现市场覆盖广度。
   - 底部再加“精选场景模板”标签带，强化行业化与模板沉淀属性。
4. **右侧部署与运维说明**
   - 上部用四步路径说明“最短上线链路”，强调预置参数和一键化部署。
   - 中部四张小卡承接生命周期、数据、监控和模板沉淀能力。
   - 底部差异化卡总结 Rune 相比普通模板商店更完整的平台闭环。
5. **底部结论条**
   - 用一句话收束“AI + 容器双场景”和“创新 + 治理平衡”的平台价值。

## 视觉风格
- **整体方向**：延续白底科技感，但相比 Slide 10 更偏“产品市场 / 模板商店”气质，引入琥珀色作为商业化强调色。
- **背景**：
  - 使用浅蓝网格作为基础秩序感。
  - 右上加入暖色柔光和轨道圆环，制造“市场活跃度”和“应用流转”氛围。
- **卡片语言**：
  - 左侧应用卡采用不同浅色渐变，模拟真实产品市场中的多类型卡片。
  - 右侧路径卡与价值卡维持更克制的企业风格，避免整页过于花哨。
  - 底部结论条回到深色品牌带，保证收束有力度。
- **品牌映射**：
  - 蓝色：平台与标准化交付
  - 绿色：数据和稳定运维
  - 紫色：AI 工作流与生成式应用
  - 琥珀色：市场化模板与业务价值

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题区从上方淡入。
2. `0.26s`：摘要条从下方淡入。
3. `0.38s`：左侧应用市场主界面整体显现。
4. `0.50s`：右侧部署路径与说明卡整体显现。
5. `0.82s`：底部结论条上浮入场。

## 交互设计
- 左侧 6 张应用卡在 `hover` 时轻微抬升，增强“可浏览的模板市场”感觉。
- 右侧生命周期卡与差异化卡也提供较轻的悬浮反馈，但弱于应用卡。
- 市场外壳和顶部指标卡保持稳定，避免削弱主界面的截图感。

## 页面目标
- 让用户快速理解 Rune 应用市场不只是“应用列表”，而是从模板发现到上线运维的一站式入口。
- 同时突出其差异化优势：统一数据管理、生命周期运维、企业专属模板沉淀，以及 AI 与容器场景的融合支持。
