# Slide 19: 灵活的交付选项

## 页面逻辑与数据映射
- **主题**：把“部署模式”改造成客户选型视角的三版本对比页，而不是架构图或连续体示意图。
- **核心观点**：Rune 提供单机体验版、集群高可用版、多集群联邦版三种部署版本，企业可以像选择产品版本一样按当前规模和治理需求做判断。
- **本页约束理解**：
  - 只继承全局白底科技感、浅网格、柔光和统一动效节奏。
  - 用户明确要求不要参考其他页面的版式，因此本页独立按“版本对比页”思路设计。

## 布局体系 (Three-Edition Compare Board)
采用 **顶部标题 + 中央三栏版本卡片 + 底部统一能力条 + 结论条**：
1. **顶部标题区**
   - 标题点出“灵活的交付选项”。
   - 副标题明确这页的阅读方式是“像选产品版本一样选部署方案”。
2. **版本对比主舞台**
   - 三张卡片严格并排。
   - 左侧是 `Starter` 对应单机体验版。
   - 中间是 `Professional` 对应集群高可用版，作为推荐生产版本强化视觉权重。
   - 右侧是 `Enterprise` 对应多集群联邦版。
3. **卡片内部结构**
   - 顶部：版本编号与适用标签。
   - 中部：大号部署形态色块，像定价页里的“核心规格”。
   - 下部：三条规格对比项，分别说明适用阶段、建设目标、升级路径。
   - 底部：胶囊能力点，作为快速扫读标签。
4. **统一能力条**
   - 单独放在三卡下方，强调三种版本共用同一能力底座。
5. **结论条**
   - 收束为“企业可以按当前 IT 现状渐进选择版本，而不是一次选错后重建平台”。

## 视觉风格
- **整体方向**：更接近企业软件官网里的“版本 / 套餐对比页”，而不是幻灯片母版式信息图。
- **背景**：
  - 维持白底、浅蓝网格、柔光科技感，但背景弱化，让注意力集中在 3 张卡片本身。
- **组件语言**：
  - 三张卡全部等宽并排，保证第一眼就是比较关系。
  - 中间卡片轻微抬高并加重阴影，承担推荐版本角色。
  - 每张卡中段使用高对比渐变规格块，强化“版本规格”识别。

## 入场动画序列
依赖 `.active` 类触发：
1. `0.12s`：标题进入。
2. `0.28s`：整块对比主舞台进入。
3. `0.40s`：左卡进入。
4. `0.52s`：中卡进入。
5. `0.64s`：右卡进入。
6. `0.74s`：统一能力条进入。
7. `0.88s`：底部结论条收尾。

## 页面目标
- 让客户在 3 秒内理解 Rune 提供的是 3 个可选部署版本。
- 让销售和方案团队能够直接对着这一页讲“你现在适合哪个版本、以后怎么升级”。
